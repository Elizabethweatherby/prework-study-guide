HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The li element is used to represent an item in a list
  • The a element followed by href is used for a link
  • The b element is used to make the text bold
  • The div element is a division or part of an HTML document
  • THe br element is used for a break
  • The h1 or any number element is used for titles
  • The i element makes an italic text
  • The img element is for images
  • The ol element is an ordered list and ul is an unordered list
  • The p element is for paragraph
  • The span element styles part of a text

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • color sets the color you want to use and the properties are RGB, hex, and keyword.
  • line-height sets the distance between lines and the values are normal, number, length, and %.
  • letter-spacing increases or decreases space between characters, values are normal, length.
  • text-align aligns the text in an element, values are left, right, center, and justify
  • text-decoration adds decoration to text, calues are nonem underline, overline, and line-through
  • text-indent indents the first line of text in an element and the values are length and %
  • text-transform controls the letters in an elementand the values are none, capitalize, uppercase, and lowercase

Git

  • git status checks what branch we are on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add moves changes from working directory to staging area
  • git branch lets you create isolated development environments in a single repo
  • git checkout lets you navigate branches, old commits and file revisions and combined with basic git comands lets you work on particular line of develpoment
  • git clean removes untracked files from working directory
  • git clone creates a copy of an existing git repo
  • git commit takes a staged snapshot and commits it to project history. combined with git add is a basic workflow for all git users
  • git commit--amend lets you amend the most recent commit
  • git fetch will download a branch from another repo along with all the associated commits and files
  • git init initializes a new git repo. if you want to place a project under revision control the is the first command
  • git log lets you explore previous revisions of a project
  • git merge lets you integrate changes from divergent branches
  • git pull downloads a branch from a remote repo then merges it into the current branch
  • git push lets you move a local branch to another repo
  • git rebase lets you move branches around to help avoid unnecessary merge commits
  • git rebase -i is used to begin an interactive rebasing session letting you add, edit, or delete commits along the way
  • git reset undoes changes to files in the working directory
  • git revert undoes a committed snapshot
